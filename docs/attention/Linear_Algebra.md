# Python 中的线性代数运用

## 1. PyTorch 中的维度操作与张量求和

在使用 PyTorch 处理张量时，维度（dimension）是最基础也最容易出错的部分之一。很多线性代数相关操作，例如求和、均值、矩阵乘法与广播，本质上都依赖于我们是否正确理解张量每一个维度的含义。

以一个形状为 `([2, 5, 4])` 的张量为例，我们可以把它理解为：
- 第 0 维大小为 `2`
- 第 1 维大小为 `5`
- 第 2 维大小为 `4`

当我们对某些维度执行 `sum()` 操作时，被指定的维度会被压缩掉；如果使用 `keepdim=True`，那么这些维度不会消失，而是保留为大小为 `1` 的维度。这在后续广播计算中非常常见，也非常实用。

### 示例代码

```python
import torch

a = torch.ones((2, 5, 4))

print(a.shape)  # torch.Size([2, 5, 4])

print(a.sum(dim=[0, 2]).shape)  # torch.Size([5])

print(a.sum(dim=[0, 2], keepdim=True).shape)  # torch.Size([1, 5, 1])
```

### 结果分析

上面的张量 `a` 一共有 3 个维度。当执行 `a.sum(dim=[0, 2])` 时，表示对第 0 维和第 2 维同时求和，只保留第 1 维，因此输出形状为 `torch.Size([5])`。

如果改成 `a.sum(dim=[0, 2], keepdim=True)`，那么第 0 维和第 2 维虽然仍然被求和，但不会被直接删除，而是保留成长度为 `1` 的维度，所以输出形状变成 `torch.Size([1, 5, 1])`。

这种保留维度的方式在神经网络中尤其常见。例如，我们经常需要先对某些维度做聚合，再与原张量进行广播计算。如果提前保留维度，后续代码会更自然，也能减少手动调用 `unsqueeze()` 的次数。

### 一个直观理解

如果把 `a` 看成一个三维数据块：
- 第 0 维可以理解为样本数或 batch 维
- 第 1 维可以理解为序列长度或中间特征维
- 第 2 维可以理解为每个位置上的特征维度

那么 `sum(dim=[0, 2])` 的含义就是：
“把第 0 维和第 2 维都累计起来，只观察第 1 维上每个位置的总和。”

而 `keepdim=True` 则表示：
“虽然完成了求和，但保留原始维度结构，方便后续继续参与张量运算。”

**总结：**
- `dim` 用来指定在哪些维度上进行聚合运算。
- 不使用 `keepdim` 时，被聚合的维度会被移除。
- 使用 `keepdim=True` 时，被聚合的维度会保留为大小为 `1` 的维度。
- 在广播、归一化和注意力机制相关计算中，保留维度通常会让代码更简洁。
